1.Qus: Discuss the scope of `var`, `let`, and `const`


Ans: In the JavaScript, we can declare to variables in var, let and const. But they are deference. In this Blog, we can explain it.

Var

Var is global scope and functional scope. We can access anywhere to var variables.

if(true){
var num = 20:
}
console.log(num); //output:20

We can redeclare and resign to var variable.

var name = "RahiL";
console.log(nam); //output:RahiL

//reassign
name = "Hello world";
console.log(nam); //output:Hello world

//redeclare
var name = "My name is RahiL";
console.log(nam); //output:My name is RahiL

Let

let is block scope. We can't access anywhere to let variables.

if(true){
let num = 20:
console.log(nam); //output:20
}
console.log(num); //output:ReferenceError: num is not define

We can reassign but can't be redeclare to let variable.


let name = "RahiL";
console.log(nam); //output:RahiL

//reassign
name = "Hello world";
console.log(nam); //output:Hello world

//redeclare
let name = "My name is RahiL";
console.log(nam); //// SyntaxError: Identifier 'number' has already been declared

const

const is block scope. We can't access anywhere to const variables. As like let variables.

if(true){
let num = 20:
console.log(nam); //output:20 }
console.log(num); //output:ReferenceError: num is not define

We can't redeclare and resign to const variable.

let name = "RahiL";
console.log(nam); //output:RahiL

//reassign
name = "Hello world";
console.log(nam); //TypeError: Assignment to constant variable.

//redeclare
let name = "My name is RahiL";
console.log(nam); //// SyntaxError: Identifier 'number' has already been declared

2. Qus: Tell us the use cases of null and undefined

Null

Null is an empty value. It's meaning in JavaScript having no value. It's is also primitive type in JavaScript.

let x = null; console.log( typeof (x)); //output:object

So, we can tell null is an object. It's like primitive data type

Undefined

Undefined means by not defined to in JavaScript.It's global object

console.log( typeof undefined); //output:undefined

so we can tell us undefined is undefined.

Many ways to we can get undefined

  • Variable that not define
  • function with no return
  • in function parameter that is not passed

3. Qus:What do you mean by REST API?

REST full meaning is Representational State Transfer and API full meaning is Application Programming Interface.

REST is a way of access web service.It's a fexible way without having any processing data. It's used to fetch or given some information of web service.

REST API uses only HTTP request. In HTTP there are five methods used in a REST based arcitecture. The five method is post, get, put, patch and delete.

  • Post: Post is used to send data to a server to create a new resource
  • Get: Get is used to request data from a resource.
  • Put: Put is used to create a new resource or replace if the resource exist.
  • Patch: Patch is used to send data to a server and update resource
  • Delete: Delete method used to the specific resource data
